2.3 关键优化详解

🔄 优化①：人工全量审核 → LLM自检 + 抽检
  • 旧：每篇15分钟，70篇=17.5小时
  • 新：用DeepSeek二次调用，验证关键数据点（技能倍率、CD、数值），只有5%抽检
  • 省时：17.5小时 → 1.5小时
  • 成本：自检调用约$0.001/篇
🆕 新增②：LLM自检验证
  • 生成后，用第二个Prompt：「以下攻略中的数值是否与原始数据一致？」
  • LLM会标记出不一致的地方
  • 只修复标记项，不是通读全文
  • 幻觉率从~5%降至~1%
🆕 新增③：SEO元数据自动生成
  • 同一次Prompt中，要求LLM输出：
  • ① SEO Title（60字符内）
  • ② Meta Description（155字符内）
  • ③ Schema.org 结构化数据
  • ④ H1/H2标签结构
  • 零额外成本，一次生成搞定
🆕 新增④：版本追踪标签
  • 每篇攻略标注：「适用于 v4.8 | 更新于 2026-06-17」
  • 游戏更新时，自动标记旧文章为「待更新」
  • 用户看到过期标签 → 信任度提升（而非看到错误数据）
  • 这是区分于「从不更新」的垃圾攻略站的关键差异
🆕 新增⑤：程序化SEO页面
  • 同一份数据 → 生成多种页面：
  • ① 角色详情页
  • ② 角色vs角色对比页
  • ③ 武器排行榜页
  • ④ 按元素的攻略索引页
  • 1份数据 → 4-10个SEO着陆页
🆕 新增⑥：邮件捕获+日更数据
  • 邮件：首页+文章底加订阅框，比Discord更早启动私域
  • 日更数据：每天自动跑一次Wiki API同步，检测数值变化
  • 联动：数据变化 → 自动更新攻略 → 发邮件通知订阅者
  • 这是「活」的攻略站，不是「死」的文章库

五、完整内容生产Pipeline（代码级设计）

5.1 一次内容生成的完整调用链

// Step 1: 获取角色数据
GET https://api.honeyimpact.com/characters/{name}
→ 返回：技能倍率、基础属性、突破材料、武器适配列表

// Step 2: 构建Prompt（模板化）
prompt = BUILD_GUIDE_TEMPLATE.format(
  character_data = step1_result,
  current_version = "4.8",
  date = "2026-06-17"
)

// Step 3: DeepSeek V4 Flash 生成攻略
response = deepseek.chat.completions.create(
  model = "deepseek-v4-flash",
  messages = [{"role": "user", "content": prompt}],
  max_tokens = 3000
)

// Step 4: DeepSeek 自检验证（同一模型，不同Prompt）
verify_prompt = VERIFY_TEMPLATE.format(
  original_data = step1_result,
  generated_guide = response
)
verify = deepseek.chat.completions.create(
  model = "deepseek-v4-flash",
  messages = [{"role": "user", "content": verify_prompt}]
)
→ 输出：不一致的数据点列表（通常0-3个）

// Step 5: 自动修复不一致
if verify.inconsistencies:
  guide = auto_fix(guide, verify.inconsistencies, original_data)

// Step 6: 生成SEO元数据（同一次调用中输出）
→ SEO title, meta description, schema.org, H1/H2结构

// Step 7: 写入MDX文件 + 版本标签
write_file(f"/content/guides/{name}.mdx",
  frontmatter = {
    title, description, version: "4.8",
    updated: "2026-06-17", schema
  },
  body = guide
)

// Step 8: 自动生成程序化SEO页面
generate_comparison_pages(name) // 对比页
generate_element_index(name) // 元素索引页
